When is software development as a group a bad idea?

  • For such simple tasks that working as a group would only slow it down and cause more problems.
  • I think a lot of the things we did would've been better to do individually

    • Describe the software development process used for the class site

    • Each week we were given a different task to be completed by the end of the week before the next week started for a different task. Students led different weeks in pairs of 2 and had to work together to think of a plan that the rest of the class could work on for that week

      • Which processes that you read about most resemble what we did? Which resembled it the least?

      • Scrum the most because Work is set to be done by the end of the week and we are like the development team and you are the overseer. Continous learning, we learned as time went on.
      • Kanban the least because work is expected to be done within a certain time period.

      What were the goals of the development process we used?

      • Get things done at a certain time so we could add them to the website
      • We learned new things as time went on, found interesting ways to do things we previously did a different way
      • To get a feel for what an actual software development job could be like

      If you're to teach the course, how would you do it?

      • Participation and attempting to try for a grade
      • Each student would individually make their own website including only their files
      • Everything would be due at the end of the year since it is not focused on one website for the whole class
      • Each week there would be something to say where each person should be at in there work to not fall behind
      • There would be no late work because it is all due at the end so you just need to be finished by the end of the semester
      • No punishment for missing class unless it is excessive. Should be expected to watch zoom recording if gone
      • Each week should focus on a topic such as a first retro and Monday, Wednesday, and Friday can be broken up into work days like we did with each pair of students leading class
      • Students would be able to upload their work to their website as they progress and that way be able to get feedback from you and other students along the way
