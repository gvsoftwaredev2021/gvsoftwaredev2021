Thoughts about software development job after class activities

We have had 2 different coding products for each individual, a calibration exercise and an HTML group project. Each product had its own features to pay attention to and reflect on for experience. The calibration exercise was to re-evaluate ourselves as a software developer with independent skills for development. The group project went further and got us to collaborate with a teammate to fulfill the goals.

After the developments, I found that I did well with some parts along the progress. First, I managed to finish the code for the individual project early, and that helped gain more time for debugging and testing. Second, I found and managed to use a platform that is easily accessible by everyone. This is beneficial in the whole. Third, I stepped up and did what was necessary when the development was needed. Last, I got along well with my teammate in the process.

However, I still have something to improve for a better preparation for a full-time development role. First, I need to collaborate more closely with my teammate to fix issues that arise earlier. Second, the accessible platform that I used cannot be used by others to create content easily, so that created difficulty and discouraged the effort to gather on this same platform. A choice for development process needs to be tested as well as the development product.
